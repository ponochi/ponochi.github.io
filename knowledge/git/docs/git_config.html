"ぽのち"のナレッジ - 【git config】

クライアントPCに git cli をインストールして、git を使い始めるときや、開発プロジェクトなどで個別に設定をチューニングする必要がある場合などに、git configコマンドで設定をする必要が出てきます。

git-config - Get and set repository or global options

When writing, the new value is written to the repository local configuration file by default, and options --system, --global, --worktree, --file can be used to tell the command to write to that location (you can say --local but that is the default).


上はgitのドキュメントからの引用ですが、git config のオプションとして、以下のものが使えることがわかります。
  • --system
  • --global
  • --worktree
  • --file
それから、上記の４つに加えて、デフォルト値として --local があります。
上から下に向けて、gitのシステム全体⇛より狭い範囲に対して、設定項目を適用できます。
個人プロジェクトなどでは、--global で、複数プロジェクトを跨いで設定項目を適用させるか、デフォルトの --local で、リポジトリ単位に対して個別に設定していくことが殆どでしょう。

次に、実際に最初に設定する例を挙げます。(とりあえず、最低限これだけ設定しておけば、あまり困ることはない)
ここでは、複数リポジトリを作成した際に、すべてに設定が適用されるよう --globalオプション を使って設定するものとします。

 $ git config --global user.name "<name>"
 $ git config --global user.email "<user@example.com>"
 $ git config --global core.editor "<editor>"
 $ git config --global color.ui "<scheme>"
 $ git config --global init.defaultBranch "<branch name>"
user.name に関しては、名と姓(英語表記なので逆)のあいだにスペースが入るため、名と姓をひと括りの値として認識できるよう、ダブルクオートで囲う必要があります。(例 "Yoshihiro Nakahara"
core.editor については、Linuxでいえば vi, vim, nvim, nano, emacsなど普段お使いのものを設定しておけば、コミットメモを書く際に指定しておいたエディタが立ち上がります。
次に color.ui は表示色の設定ですが、一般的には auto をスキームとして設定しておくと、gitがいい具合に色分けしてくれます。が、auto の色分けが気に食わん！という方は、スキームを指定する必要があります。(ここでは個別の設定は取り上げないので、ドキュメント等で調べて設定してください。)
最後の init.defaultBranch に関しては、過去にはデフォルトのブランチ名は master だったのですが、 master(主人) / slave(奴隷) を連想させるということで、デフォルトのブランチ名が main になるような変更が入りました。この項目を設定しておかないと、コミットしようとした際に警告が出るので、最初に設定しておきましょう。また、大規模プロジェクトなどで main 以外のデフォルトブランチ名が指定されている場合は、--global ではなく、--local (デフォルトなので、実際のオプションとして --local は指定しない) で設定しておけば大丈夫でしょう。
